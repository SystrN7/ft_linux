Introduction à enchant

Le paquet enchant fournit une interface générique avec diverses bibliothèques existantes de vérification d'orthographe.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.4.

Installation de enchant

Installez enchant en lançant les commandes suivantes : 

./configure --prefix=/usr --disable-static &&
make

Pour lancer les tests, unittest-cpp doit être installé et l'option --enable-relocatable doit être passée à configure ci-dessus. Si ces conditions sont respectées, les tests peuvent être lancés avec make check.

Maintenant, en tant qu'utilisateur root : 

make install                                   &&
rm -rf /usr/include/enchant                    &&
ln -sfv enchant-2       /usr/include/enchant   &&
ln -sfv enchant-2       /usr/bin/enchant       &&
ln -sfv libenchant-2.so /usr/lib/libenchant.so &&
ln -sfv enchant-2.pc    /usr/lib/pkgconfig/enchant.pc

Explication des commandes

--disable-static : Ce paramètre empêche l'installation des versions statiques des bibliothèques.

rm -rf /usr/include/enchant; ln -sfn ... : Crée des liens symboliques pour cette version du paquet pour que d'autres paquets puisse le trouver avec l'ancien nom.

Configuration d'enchant

Fichiers de configuration

~/.enchant et /usr/share/enchant/enchant.ordering

Informations de configuration

Vous pouvez tester votre installation et votre configuration en créant un fichier test et en lançant les commandes suivantes (vous pouvez remplacer le dictionnaire en_GB par n'importe quel autre que vous avez téléchargé à l'installation de Aspell-0.60.6.1) : 

cat > /tmp/test-enchant.txt << "EOF"
Tel me more abot linux
Ther ar so many commads
EOF

enchant -d en_GB -l /tmp/test-enchant.txt &&
enchant -d en_GB -a /tmp/test-enchant.txt

Vous verrez une liste des mots mal orthographié suivi par une liste des alternatives pour eux.

Voir plus de détails dans la page de manuel d'enchant.
