xinit-1.4.0

Introduction à xinit

Le paquet xinit contient un script utilisable pour démarrer xserver.

Ce paquet ne fait pas partie de Xorg Katamari et est seulement fourni comme dépendance d'autres paquets ou pour tester l'installation complète de Xorg.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.4.

Installation de xinit

Commencez par corriger startx, afin que la session X démarre sur le premier terminal virtuel inutilisé, qui est normalement vt7 : 

sed -e '/$serverargs $vtarg/ s/serverargs/: #&/' \
    -i startx.cpp

Note

Si vous avez déjà installé ce paquet sans le sed précédent, ce n'est pas utile de le réinstaller. Modifiez juste le startx installé, en tapant en tant qu'utilisateur root : sed -i '/$serverargs $vtarg/ s/serverargs/: #&/' /usr/bin/startx

Installez xinit en lançant les commandes suivantes : 

./configure $XORG_CONFIG --with-xinitdir=/etc/X11/app-defaults &&
make

Ce paquet n'est pas fourni avec une suite de tests.

Maintenant, en tant qu'utilisateur root : 

make install &&
ldconfig
