libtirpc-1.1.4

Introduction à libtirpc

Le paquet libtirpc contient des bibliothèques qui supportent des programmes utilisant l'API de Remote Procedure Call (RPC). Il remplace le RPC, mais pas les entrées de la bibliothèque NIS qui se trouvaient dans glibc.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.4.

Installation de libtirpc

Note

Si vous mettez à jour ce paquet, vous devrez aussi mettre à jour toute version existante de rpcbind-1.2.5. 

./configure --prefix=/usr                                   \
            --sysconfdir=/etc                               \
            --disable-static                                \
            --disable-gssapi                                &&
make

Ce paquet n'est pas fourni avec une suite de tests.

Maintenant, en tant qu'utilisateur root : 

make install &&
mv -v /usr/lib/libtirpc.so.* /lib &&
ln -sfv ../../lib/libtirpc.so.3.0.0 /usr/lib/libtirpc.so

Explication des commandes

--disable-static : Ce paramètre empêche l'installation des versions statiques des bibliothèques.

--disable-gssapi : Ce paramètre est utile si aucun GSSAPI est installé. Effacez ce paramètre si vous en avez un d'installé (par exemple MIT Kerberos V5-1.17) et que vous souhaitiez l'utiliser.

mv -v /usr/lib/libtirpc.so.* ... : Déplace les bibliothèques partagées dans /lib pour qu'elles soient disponibles avant que /usr ne soit monté.
