DConf-0.30.1 / DConf-Editor-3.30.2

Introduction à Dconf

Le paquet Donf contient un système de configuration de bas niveau. Son but principal est d'offrir une fondation à  GSettings sur des plate-formes qui n'ont pas déjà de systèmes de stockage de configuration.

DConf-Editor, comme son nom le suggère, est un éditeur graphique pour la base de données DConf. L'installation est optionnelle, car gsettings de GLib-2.58.3 fournit une fonctionnalité similaire en ligne de commandes.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.4.

Installation de DConf

Installez DConf en lançant les commandes suivantes : 

mkdir build &&
cd    build &&

meson --prefix=/usr --sysconfdir=/etc -Dbash_completion=false .. &&
ninja

Ce paquet n'est pas fourni avec une suite de tests.

En tant qu'utilisateur root : 

ninja install &&
cp gsettings/libdconfsettings.so /usr/lib/gio/modules/

Maintenant, installez facultativement l'éditeur: 

cd ..              &&
tar -xf ../dconf-editor-3.30.2.tar.xz &&
cd dconf-editor-3.30.2                &&

mkdir build &&
cd    build &&

meson --prefix=/usr --sysconfdir=/etc .. &&
ninja

En tant qu'utilisateur root : 

ninja install

Explication des commandes

-Denable-gtk-doc : Utilisez ce paramètre si GTK-Doc est installé et que vous souhaitez reconstruire la documentation de l'API.

Répertoires installés: /usr/{include,share/gtk-doc/html}/dconf

dconf-editor

libdconf.so

contient les fonctions de l'API de DConf.
