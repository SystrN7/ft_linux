GPGME-1.12.0

Introduction à GPGME

Le paquet GPGME est une bibliothèque C qui permet d'ajouter le support du chiffrement à un programme. Il est conçu pour faciliter l'accès pour les applications à des moteurs de chiffrement de clés tels que GnuPG ou GpgSM. GPGME fournit une API de chiffrement de haut niveau pour le chiffrement, le déchiffrement, l'authentification, la vérification de signature et la gestion de clé.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.4.

Installation de GPGME

Installez GPGME en lançant les commandes suivantes : 

./configure --prefix=/usr --disable-gpg-test &&
make

Pour tester les résultats, vous devriez avoir installé GnuPG-2.2.13 et supprimé le paramètre --disable-gpg-test ci-dessus. Lancez : make check.

Maintenant, en tant qu'utilisateur root : 

make install

Explication des commandes

--disable-gpg-test : si ce paramètre n'est pas passé à configure, les programmes de tests seront construits pendant l'étape make, ce qui demande GnuPG-2.2.13. Ce paramèter n'est pas requis si GnuPG-2.2.13 est installé.

Contenu

Programme installé: gpgme-config et gpgme-tool
Bibliothèques installées: libgpgme-pthread.so et libgpgme.so
Répertoire installé: /usr/share/common-lisp/source/gpgme

Descriptions courtes

gpgme-config

est utilisé pour obtenir des informations de compilation et d'édition de liens sur GPGME.

gpgme-tool

affiche l'empreinte et l'id de la clé avec les serveurs de clés.

libgpgme-pthread.so

contient les fonctions de l'API GPGME pour les applications utilisant pthread.

libgpgme.so

contient les fonctions de l'API GPGME.
