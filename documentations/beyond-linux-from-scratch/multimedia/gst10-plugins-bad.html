Introduction à GStreamer Bad Plug-ins

Le paquet GStreamer Bad Plug-ins contient un ensemble de greffons qui ne sont pas à la hauteur comparé avec le reste. Ils peuvent être presque de bonne qualité, mais il leur manque quelque chose - que ce soit une bonne révision du code, de la documentation, un ensemble de tests, un mainteneur réel direct, ou une réelle utilisation à grande échelle.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.4.

Installation de GStreamer Bad Plug-ins

Note

Si vous voulez un greffon pour une dépendance donnée, cette dépendance doit être installée avant cette application.

Appliquez un correctif pour permettre la construction avec libvpx-1.8. 

patch -Np1 -i ../gst-plugins-bad-1.14.4-fdkaac_2-1.patch

Installez GStreamer Bad Plug-ins en lançant les commandes suivantes : 

./configure --prefix=/usr                                           \
            --disable-wayland                                       \
            --disable-opencv                                        \
            --with-package-name="GStreamer Bad Plugins 1.14.4 BLFS" \
            --with-package-origin="http://www.linuxfromscratch.org/blfs/view/svn/" &&
make

Pour tester les résultats, lancez : make check. Plusieurs tests ont besoin d'un émulateur de terminal dans une session graphique. Un test, dash_mpd, est connu pour échouer.

Maintenant, en tant qu'utilisateur root : 

make install

Explication des commandes

--disable-wayland : Ce paramètre est requis en l'absence d'un environnement de bureau complet basé sur wayland.

--disable-opencv : Ce paramètre est requis si opencv-4.0.1 est installé sans module supplémentaire (en particulier bgsegm).
