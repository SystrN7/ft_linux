Phonon-backend-gstreamer-4.9.0

Introduction à Phonon-backend-gstreamer

Ce paquet fournit un moteur Phonon qui utilise le framework média GStreamer.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.4.

Informations sur le paquet

  • Téléchargement (HTTP) : http://download.kde.org/stable/phonon/phonon-backend-gstreamer/4.9.0/phonon-backend-gstreamer-4.9.0.tar.xz

  • Téléchargement (FTP) :

  • Somme de contrôle MD5 du téléchargement : b2b0ca327908948ce8cdba117e31bf80

  • Taille du téléchargement : 80 Ko

  • Estimation de l'espace disque requis : 4.5 Mo

  • Estimation du temps de construction : 0.4 SBU

Dépendances de Phonon-backend-gstreamer

Requises

gstreamer-1.14.4, libxml2-2.9.9 et phonon-4.10.2

Recommandées

gst-plugins-base-1.14.4 (nécessaire pour la sortie sur ALSA), gst-plugins-good-1.14.4 (nécessaire pour la sortie sur PulseAudio), gst-plugins-bad-1.14.4 (nécessaire pour le support AAC/M4A) et gst-plugins-ugly-1.14.4 (nécessaire pour le support MP3).

Notes utilisateur : http://wiki.linuxfromscratch.org/blfs/wiki/phonon-backend-gstreamer

Installation de Phonon-backend-gstreamer

Note

Ce paquet se décompresse dans le répertoire phonon-gstreamer-4.9.0.

Installez Phonon-backend-gstreamer en lançant les commandes suivantes : 

mkdir build &&
cd    build &&

cmake -DCMAKE_INSTALL_PREFIX=/usr    \
      -DCMAKE_BUILD_TYPE=Release     \
      -DPHONON_BUILD_PHONON4QT5=ON   \
      -Wno-dev .. &&
make

Maintenant, en tant qu'utilisateur root : 

make install

Explication des commandes

-DCMAKE_BUILD_TYPE=Release : Ce paramètre est utilisé pour appliquer un plus haut niveau d'optimisation du compilateur.

-DPHONON_BUILD_PHONON4QT5=ON : Ce paramètre est utilisé pour assurer que la version qt5 du moteur est construite même si Qt4 est présent.

Contenu

Programmes installés: aucun
Bibliothèques installées: phonon_gstreamer.so
Répertoires installés: /usr/lib/qt5/plugins/phonon4qt5_backend
