lxqt-qtplugin-0.12.0

Introduction à lxqt-qtplugin

Le paquet lxqt-qtplugin fournit un greffon d'intégration à la plateforme Qt pour LXQT.

Ce paquet est connu pour se construire correctement sur une plateforme LFS-8.2.

Informations sur le paquet

  • Téléchargement (HTTP) : https://github.com/lxde/lxqt-qtplugin/releases/download/0.12.0/lxqt-qtplugin-0.12.0.tar.xz

  • Téléchargement (FTP) :

  • Taille du téléchargement : 23 Ko

  • Estimation de l'espace disque requis : 2.6 Mo

  • Estimation du temps de construction : 0.1 SBU

Dépendances de lxqt-qtplugin

Requises

liblxqt-0.12.0 et libdbusmenu-qt-0.9.3+16.04.20160218

Notes utilisateur : http://wiki.linuxfromscratch.org/blfs/wiki/lxqt-qtplugin

Installation de lxqt-qtplugin

Installez lxqt-qtplugin en lançant les commandes suivantes : 

mkdir -v build &&
cd       build &&

cmake -DCMAKE_BUILD_TYPE=Release          \
      -DCMAKE_INSTALL_PREFIX=$LXQT_PREFIX \
      ..       &&

make

Ce paquet n'est pas fourni avec une suite de tests.

Maintenant, en tant qu'utilisateur root : 

make install

Configuration de lxqt-qtplugin

Informations sur la configuration

Pour utiliser le greffon dans Qt 5, la variable d'environnement QT_QPA_PLATFORMTHEME doit être initialisée à « lxqt ». Une façon de faire est de lancer en tant qu'utilisateur root : 

cat >> /etc/profile.d/lxqt.sh << "EOF"
# Begin lxqt-qtplugin configuration

export QT_QPA_PLATFORMTHEME=lxqt

# End lxqt-qtplugin configuration
EOF

Contenu

Répertoire installé: $QT5DIR/plugins/platformthemes (si un Qt5 minimal a été construit comme expliqué dans pré-installation du bureau LXQt)
